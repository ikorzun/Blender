# Приёмка 3D-ассетов (зона ГРАФИКА)

Модели владелец складывает в **`3d assets/`** в корне рабочей папки
(`funnel-game-v1/3d assets/`). Папка НЕ в гите (untracked) и её нет в
воркдеревьях — аудит запускать из основной папки.

Перед интеграцией КАЖДАЯ модель прогоняется через аудит:

```bash
node tools/audit-glb.js "3d assets"                 # геометрия, материалы, построение
node tools/extract-glb-textures.js "3d assets" /tmp/tex   # вынуть PNG и найти общие атласы
```

## Требования

Игра — однофайловый `index.html` без рантайм-лоадеров: модель
конвертируется **на этапе разработки** в data-модуль `src/app/NN-имя.js`
(как `35-steak.js`). Отсюда жёсткие рамки.

| Что | Требование | Почему |
|---|---|---|
| Полигонаж | **≤ 400 тр.** (эталон — стейк, 144) | на экране до 181 предмета: 400 × 181 ≈ 72k тр. |
| Примитивы | **1** на модель | каждый лишний — отдельный draw call ×181 |
| Цвет | **вершинные цвета** (COLOR_0) либо плоские Kd | текстур в пайплайне нет; вуаль недоступности умножается на цвет |
| Origin | центр в нуле, охват нормализован к `rc` | конвертер делает сам |
| Стороны | **FrontSide** | `doubleSided` удваивает фрагментную работу |
| Прозрачность | **OPAQUE**; если нужна — только `opacity` | BLEND ломает сортировку в плотной куче; `transmission` ЗАПРЕЩЁН (−55% кадра) |
| Плоские модели | указать `wr` в TYPES | охватный радиус переоценивает ширину → шторм ложных спасений у стены |
| Вогнутость | заполненность ≥ ~0.25 | ниже — convex hull врёт, нужен ручной компаунд (как тор/чайник) |
| Сжатие | без Draco / meshopt / basisu | декодер = внешний wasm, однофайловость ломается |
| Анимации/скины | нет | не поддерживаются |

| модель | тр. | вес | вердикт |
|---|---|---|---|

| `CellPhone_Retro` | 574 | 384 КБ | полигонаж ×1.4 от лимита, 2 примитива, плоская (0.20) |

### Общее для всех пяти

1. **Вершинных цветов нет ни у одной** — цвет держится на текстурах
   (`baseColorTexture`). Это главное расхождение с пайплайном.
2. **Все `doubleSided`** — снять.
3. **Все смещены от нуля** (у `CellPhone` центр на Y=1.41) — конвертер выправит.
4. `NoelCap` и `Present01` — `alphaMode: BLEND` из-за альфы в атласе; перевести
   в OPAQUE, иначе прозрачность сломает сортировку в куче.
5. `CellPhone` и `Mouse` — по 2 примитива, нужен merge.
6. Расширения `KHR_materials_specular/ior` — конвертацией отбрасываются, не блокер.

### Что делать с текстурами

Атласы у моделей **общие**: `Trim_Vapor_Column_02` делят `CellPhone` и `Mouse`,
`Atlas_02_Xmas` — `NoelCap` и `Present01` (побайтово одинаковые файлы).
По характеру это **трим-листы из плоских цветовых блоков**, полос и клеток —
такое запекается в вершинные цвета почти без потерь (потеряются только тонкие
полоски и клетка, которые на предмете ~1 юнита всё равно не читаются).

Исключение — `CA_Head`: его текстура это **лист наклеек 2048×2048** с
иллюстрациями и логотипами. В вершинные цвета не сводится; нужен либо
настоящий атлас в рантайме, либо отказ от модели.

**Рекомендация:** запекать UV→вершинные цвета для четырёх «трим»-моделей
(ноль байт текстур, пайплайн не меняется), `CA_Head` отложить.

⚠️ Модели выглядят как ассет-пак стороннего автора (в листе наклеек есть
чужой логотип «PYRE PALS»). Перед публикацией проверить лицензию.

## Интеграция 2026-07-20 (10 моделей, текстуры сняты)

Конвертер `tools/glb2module.py` -> `src/app/36-models.js` (372 КБ).
Владелец попросил «убрать все текстуры» — берём только геометрию, цвет
из палитры TYPES. Модели поставлены В НАЧАЛО TYPES, иначе на первых
уровнях их не увидеть (genLevel берёт первые typesCount типов).

**Пропущена `RetroComputerBooth.glb`** — 29 721 тр. и 23 МБ: это декорация,
а не предмет в кучу. Потолок конвертера `MAX_TRIS = 3000`, пропуски печатаются.

Замеры на уровне 1 (129 предметов), примитивы -> модели:

| метрика | было | стало | норма |
|---|---|---|---|
| topY (заполнение) | 6.95 | **4.79** | 7.5-9.0 на полном уровне |
| время до штиля | мгновенно | **~3 с** | — |
| спасений у стены | 0 | **2** | 0 |
| maxWallExcess | 0.101 | **0.165** | ≤ ~0.15 |
| вес index.html | 3.01 МБ | 3.39 МБ | ~1 МБ gzip |

Причина недозаполнения: модели тонкие и вытянутые — при равном охвате их
объём вдвое меньше шара. Охват уже поднят с 0.78 до **1.0** (`RC` в
конвертере и `rc` в TYPES держать синхронными), это подняло topY с ~3.4 до
4.79. Дальше — либо ещё крупнее, либо больше пар, но **PAIRS менять только
по решению владельца** (заполнение стакана завязано на объём, см. CLAUDE.md).

## Прореживание 2026-07-20 (вторая партия, 15 моделей)

Владелец добавил «иконочные» ассеты на 84-509 тыс. треугольников — в игру
такие не влезают физически. В конвертер добавлено **прореживание схлопыванием
вершин в сетку** (`cluster`/`decimate`), планка `TARGET_TRIS = 500`. Метод
выбран вместо честного edge-collapse осознанно: линейный по числу
треугольников (edge-collapse на полумиллионе рёбер в питоне не считается),
а огранка совпадает с нашим flat-шейдингом. Весь прогон — 7 с.

Модуль ПОХУДЕЛ: 372 КБ (10 моделей) -> 292 КБ (15 моделей).

**НЕ ВЗЯТЫ:**
- `Money 20.fbx` — формат FBX не поддержан, нужен экспорт в .glb.
- `Gold Stone.glb` — 132 байта, внутри только `{"scenes":[{"name":"Scene"}]}`
  без нод и мешей. **Пустой экспорт из Blender**, геометрии в файле нет.

**⚠️ ГЛАВНОЕ ОГРАНИЧЕНИЕ МЕТОДА:** схлопывание в сетку губит ТОНКУЮ И ПОЛУЮ
геометрию — стенки схлопываются, модель распадается на осколки. Чем больше
кратность прореживания, тем хуже. Итог по узнаваемости:

| результат | модели |
|---|---|
| читается | present01, sausage, lifebuoy, pretzel, rock01 |
| сносно | mouseretro, cellphoneretro, cahead, noelcap, concretemixer |
| **развалилось** | tallgoldencrown (84k->328, заполненность 0.03 — рассыпалась на осколки), iceskate (509k->364), retrocomputerbooth (30k->449), squaresquidart, frogaxonart |

Развалившимся нужен либо Decimate-модификатор в Blender (collapse сохраняет
форму несравнимо лучше), либо изначально низкополигональный исходник.

⚠️ Побочный эффект для тестов: `test.js` форсировал тупик после
фиксированной паузы 600 мс. Примитивы оседали мгновенно и это работало;
модели оседают ~3 с, и пока куча движется, `updateMatchRadius` каждый тик
перезаписывает форсированный `matchRadius` — тупик не наступал вовсе.
Пауза заменена ожиданием УСТОЙЧИВОГО штиля (серия спокойных опросов: сразу
после `skipIntro` бывает краткий ложный штиль ~150 мс).
