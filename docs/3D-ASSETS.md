# Приёмка 3D-ассетов (зона ГРАФИКА)

Модели владелец складывает в **`3d assets/`** в корне рабочей папки
(`funnel-game-v1/3d assets/`). Папка НЕ в гите (untracked) и её нет в
воркдеревьях — аудит запускать из основной папки.

Перед интеграцией КАЖДАЯ модель прогоняется через аудит:

```bash
node tools/audit-glb.js "3d assets"                 # геометрия, материалы, построение
node tools/extract-glb-textures.js "3d assets" /tmp/tex   # вынуть PNG и найти общие атласы
```

## Требования

Игра — однофайловый `index.html` без рантайм-лоадеров: модель
конвертируется **на этапе разработки** в data-модуль `src/app/NN-имя.js`
(как `35-steak.js`). Отсюда жёсткие рамки.

| Что | Требование | Почему |
|---|---|---|
| Полигонаж | **≤ 400 тр.** (эталон — стейк, 144) | на экране до 181 предмета: 400 × 181 ≈ 72k тр. |
| Примитивы | **1** на модель | каждый лишний — отдельный draw call ×181 |
| Цвет | **вершинные цвета** (COLOR_0) либо плоские Kd | текстур в пайплайне нет; вуаль недоступности умножается на цвет |
| Origin | центр в нуле, охват нормализован к `rc` | конвертер делает сам |
| Стороны | **FrontSide** | `doubleSided` удваивает фрагментную работу |
| Прозрачность | **OPAQUE**; если нужна — только `opacity` | BLEND ломает сортировку в плотной куче; `transmission` ЗАПРЕЩЁН (−55% кадра) |
| Плоские модели | указать `wr` в TYPES | охватный радиус переоценивает ширину → шторм ложных спасений у стены |
| Вогнутость | заполненность ≥ ~0.25 | ниже — convex hull врёт, нужен ручной компаунд (как тор/чайник) |
| Сжатие | без Draco / meshopt / basisu | декодер = внешний wasm, однофайловость ломается |
| Анимации/скины | нет | не поддерживаются |

| модель | тр. | вес | вердикт |

| `CellPhone_Retro` | 574 | 384 КБ | полигонаж ×1.4 от лимита, 2 примитива, плоская (0.20) |

### Общее для всех пяти

1. **Вершинных цветов нет ни у одной** — цвет держится на текстурах
   (`baseColorTexture`). Это главное расхождение с пайплайном.
2. **Все `doubleSided`** — снять.
3. **Все смещены от нуля** (у `CellPhone` центр на Y=1.41) — конвертер выправит.
4. `NoelCap` и `Present01` — `alphaMode: BLEND` из-за альфы в атласе; перевести
   в OPAQUE, иначе прозрачность сломает сортировку в куче.
5. `CellPhone` и `Mouse` — по 2 примитива, нужен merge.
6. Расширения `KHR_materials_specular/ior` — конвертацией отбрасываются, не блокер.

### Что делать с текстурами

Атласы у моделей **общие**: `Trim_Vapor_Column_02` делят `CellPhone` и `Mouse`,
`Atlas_02_Xmas` — `NoelCap` и `Present01` (побайтово одинаковые файлы).
По характеру это **трим-листы из плоских цветовых блоков**, полос и клеток —
такое запекается в вершинные цвета почти без потерь (потеряются только тонкие
полоски и клетка, которые на предмете ~1 юнита всё равно не читаются).

Исключение — `CA_Head`: его текстура это **лист наклеек 2048×2048** с
иллюстрациями и логотипами. В вершинные цвета не сводится; нужен либо
настоящий атлас в рантайме, либо отказ от модели.

**Рекомендация:** запекать UV→вершинные цвета для четырёх «трим»-моделей
(ноль байт текстур, пайплайн не меняется), `CA_Head` отложить.

⚠️ Модели выглядят как ассет-пак стороннего автора (в листе наклеек есть
чужой логотип «PYRE PALS»). Перед публикацией проверить лицензию.
