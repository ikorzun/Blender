# Каденция рекламы по площадкам — ЧЕРНОВИК МЕХАНИЗМА

⚠️ **СТАТУС: ЧЕРНОВИК. Поведение игры НЕ меняет, кода в `src/` не трогает,
`INTER_*` в 00-config НЕ тронуты.** Написан заранее (разрешение диспетчера
2026-07-23), чтобы к моменту «делай» от владельца обсуждать готовую
структуру, а не начинать с нуля. Зона: ИНТЕГРАЦИЯ И ПУБЛИКАЦИЯ.

⚠️ **ЧИСЕЛ ПЛОЩАДОК ЗДЕСЬ НЕТ НАМЕРЕННО.** Требования Poki и CrazyGames
берутся из их официальной документации на этапе чек-листов (п.4 блока
ИНТЕГРАЦИЯ) — по памяти такие вещи писать нельзя, политики площадок
меняются, а расхождение = отказ на ревью. Ниже проектируется МЕХАНИЗМ,
числа подставляются в него позже.

## 1. Зачем

Сейчас каденция одна на всех, захардкожена двумя константами
(`00-config.js:84-85`) и применяется в единственной точке —
`maybeInterstitial` (`78-ads.js:99-110`):

```
INTER_MIN_WINS = 2;      // не раньше 2-й победы сессии
INTER_GAP_MS   = 180000; // и не чаще раза в 3 минуты
```

Вердикт аудита плана v1: у площадок требования к межстраничной рекламе
разные, одна каденция на всех либо нарушит правила строгой площадки, либо
недоберёт дохода на либеральной. Вынести в per-platform конфиг.

## 2. Что уже проверено (факты, не предположения)

- **Обе целевые площадки поддержаны нашим SDK.** В завендоренном
  `playgama-bridge.js` (v2.0.0) есть `PLATFORM_ID.POKI = "poki"` и
  `PLATFORM_ID.CRAZY_GAMES = "crazy_games"` — рядом с `PLAYGAMA`,
  `YANDEX`, `GAME_DISTRIBUTION`, `MOCK`. Значит развилка по
  `bridge.platform.id` — правильный механизм, отдельные сборки под
  площадку не нужны.
- **Идентификатор доступен в рантайме:** `bridge.platform.id` (в SDK —
  геттер над `platformId`). Замер на живом Pages: отдаёт `mock`.
- **Гейт ровно один** — `maybeInterstitial`; все показы межстраничной
  идут через него. Разводить логику по вызовам не придётся.
- **В stub-режиме interstitial не показывается вообще** (существующий
  инвариант: «в стабе не раздражаем») — это поведение сохраняется, оно же
  защищает file:// и локальные прогоны.

## 2-бис. ⚠️ ПОСЫЛКА ЧЕРНОВИКА ТРЕБУЕТ РАЗВОРОТА (сверка с источниками 2026-07-23)

Вердикт аудита звучал как «у площадок каденция строже, вынести в конфиг».
Сверка с официальной документацией показала **обратное: Poki и CrazyGames
управляют частотой САМИ, а от игры ждут сигналов о возможностях**.

- Poki, раздел требований: «Do not implement internal ad timers» — прямым
  текстом просят НЕ держать свой таймер и полагаться на их систему.
- CrazyGames: частоту межстраничной SDK регулирует сам, включая
  взаимодействие с rewarded (их собственный потолок задокументирован на
  странице требований — числа берём оттуда, не отсюда).

Наши `INTER_MIN_WINS`/`INTER_GAP_MS` — это ровно «internal ad timer».
Значит на этих площадках правильная настройка не «сделать строже», а
**отключить наш throttle и звать показ на каждом естественном разрыве**;
площадка сама решит, показывать ли. Иначе мы своей же рукой режем показы
(и доход), формально ничего не нарушая.

Практический вывод для §3: ручка `enabled` из «площадка запрещает рекламу»
превращается в **«кто владеет темпом — мы или площадка»**, и это её
основное назначение. Структура таблицы остаётся годной, меняется смысл
значений по умолчанию для poki/crazy_games.

## 3. Предлагаемая структура (НЕ применена)

Таблица в `00-config.js` (по правилу зон ИНТЕГРАЦИЯ может ДОБАВЛЯТЬ свои
константы; существующие `INTER_*` при этом остаются как значения
по умолчанию — принятие черновика не меняет поведение ни на одной
площадке, пока таблица не заполнена):

```js
// Каденция межстраничной рекламы по площадкам. Ключ — bridge.platform.id.
// default обязан совпадать с текущим поведением: неизвестная площадка
// получает то же, что сегодня получают все.
const AD_CADENCE = {
  default:       { enabled: true, minWins: INTER_MIN_WINS, gapMs: INTER_GAP_MS },
  poki:          { /* из официальных требований Poki */ },
  crazy_games:   { /* из официальных требований CrazyGames */ },
};
```

Разрешение — чистая функция рядом с гейтом (`78-ads.js`):

```js
function cadence(){
  const id = (window.bridge && window.bridge.platform && window.bridge.platform.id) || '';
  return Object.assign({}, AD_CADENCE.default, AD_CADENCE[id] || {});
}
```

`maybeInterstitial` меняется на три строки: `enabled`, `minWins`, `gapMs`
берутся из `cadence()` вместо констант напрямую. Merge поверх `default`
означает, что площадке достаточно переопределить ОДНУ ручку.

**Ручки — ровно три, и ни одной сверх.** `enabled` нужен, потому что
площадка может запрещать межстраничную вовсе или требовать свой
API-обёртку; `minWins` и `gapMs` уже существуют. Кандидаты вроде «пауза
после первого запуска», «кап показов за сессию», «не показывать в первые
N минут» НЕ вводить умозрительно — только если конкретное требование
конкретной площадки этого потребует, и тогда одной ручкой за раз.

## 4. Как это проверять

Расширить bridge-секцию сьюта — стенд уже готов и работает: локальный
http-сервер + ПОДДЕЛЬНЫЙ `playgama-bridge.js` (появился 2026-07-23 ради
фикса облачного сейва). Достаточно параметризовать мок по `platform.id`
и ассертить, что `cadence()` вернул ожидаемые значения для `poki`,
`crazy_games` и неизвестной площадки (последняя обязана дать `default`).

⚠️ Ассерт «неизвестная площадка = сегодняшнее поведение» обязателен: он
стережёт от тихого изменения каденции там, где мы её менять не собирались.

**Телеметрия:** событие `inter` (79-telemetry) должно нести `platform` и
разрешённую каденцию — иначе соблюдение правил площадки нечем подтвердить
постфактум, а при жалобе ревью это первое, что спросят.

## 5. Открытые вопросы (в п.4, до заполнения таблицы)

1. Требования Poki и CrazyGames по межстраничной: минимальный интервал,
   допустимые моменты показа, обязательные пре-роллы/«commercial break».
   Наш показ идёт по кнопкам «Next»/«Retry» — это естественный разрыв, но
   формулировки площадок надо сверить дословно.
2. Требуют ли они СВОЙ вызов вместо общего `showInterstitial()` — Bridge
   абстрагирует, но если площадка ждёт явного сигнала «загрузка началась
   / игра возобновилась», это может быть отдельным пунктом интеграции,
   не покрываемым каденцией.
3. Rewarded-каденция: сейчас капы наши собственные (3 бесплатные встряски
   + 2 за рекламу). Есть ли у площадок ограничения на частоту rewarded —
   проверить; в эту же таблицу при необходимости.
4. Кто источник истины по числам — документация площадки или ответ их
   ревью. Расхождения фиксировать здесь же.

## 6. Источники (проверены открытием страниц 2026-07-23)

Список «куда смотреть», а не выписка требований — числа и точные
формулировки берутся отсюда на этапе чек-листов (п.4 блока ИНТЕГРАЦИЯ).
Ссылки проверены живьём; структура доков у площадок меняется, при
расхождении верить странице, а не этому файлу.

**Poki**
- `https://sdk.poki.com/new-requirements` — требования к игре, раздел
  рекламной интеграции: частота (их система), поведение игры во время
  ролика (звук, ввод), правила rewarded, политика при адблоке.
- `https://sdk.poki.com/html5` — интеграция HTML5-игры: сигнатуры
  `commercialBreak()` / `rewardedBreak()`, порядок вызовов.

**CrazyGames**
- `https://docs.crazygames.com/requirements/ads/` — требования к рекламе:
  когда допустим midgame, обязательная альтернатива для rewarded,
  поведение во время ролика. ⚠️ Их предупреждение дословно: игру
  отклоняют без объяснений, если требования не соблюдены.
- `https://docs.crazygames.com/sdk/video-ads/` — API видеорекламы:
  типы midgame/rewarded, события `adFinished` / `adError`.
- `https://docs.crazygames.com/resources/ad-monetization-guide/` —
  рекомендации по размещению (не требования, но полезно для мест показа).

**Наш слой**
- `github.com/playgama/bridge` — чем Bridge накрывает эти площадки
  (`PLATFORM_ID.POKI`, `CRAZY_GAMES` есть в завендоренной v2.0.0).
  ⚠️ ОТКРЫТО: покрывает ли Bridge требования площадок ПОЛНОСТЬЮ (мьют
  звука, пауза, блок ввода) или это остаётся на игре — проверять
  чтением его адаптеров, не предположением.

## 7. ⚠️ РИСКИ СООТВЕТСТВИЯ, ВИДНЫЕ УЖЕ СЕЙЧАС

Обнаружены при сверке источников с нашим кодом. Это НЕ чек-лист (он в
п.4) и НЕ задачи — это то, что нельзя оставить незаписанным, потому что
два пункта ведут к отказу на ревью, а один упирается в решение владельца.

| Требование площадок | У нас сейчас | Статус |
|---|---|---|
| Глушить звук игры на время ролика (обе площадки) | НЕ глушим: в `78-ads.js` мьюта нет, в `75-audio.js` только глобальный тумблер `CFG.sound` | открыто |
| Пауза игры и блок ввода на время ролика (обе) | Ввод глушится только когда виден НАШ оверлей-заглушка (`90-input.js:233`), а в bridge-режиме `showRewarded` его не показывает — во время настоящего ролика канвас живой; игра не ставится на паузу, тикает только `stats.lastAction`, чтобы миксер не ел предметы | открыто |
| Для rewarded обязательна АЛЬТЕРНАТИВА (CrazyGames — прямым текстом; Poki — «обычная» кнопка рядом с рекламной) | Альтернатива есть в коде (встряска за монеты), но `COINS_ENABLED=false` (00-config:78) — сейчас у rewarded-встряски альтернативы НЕТ | ⚠️ упирается в решение владельца (монеты скрыты его спекой) |
| При адблоке награду не выдавать | Пути `settleFail` есть; отдельной детекции адблока нет — полагаемся на состояние платформы | проверить на смоуке |
| Своё сообщение при неудаче показа | `toast('Ad unavailable')` (78-ads.js:38) | проверить: у Poki коммуникацию про адблок ведёт площадка |

Ни одна строка отсюда НЕ чинится до слова владельца: правки трогают
звук, ввод и экономику — чужие зоны и его решения.

## 8. Чего этот черновик НЕ делает

Не меняет `INTER_*`, не добавляет код в `src/`, не трогает баланс и
правила игры. Принятие = отдельная задача с прогоном сьюта и сдачей
по протоколу; до слова владельца — только этот документ.
